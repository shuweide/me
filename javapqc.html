Shor 演算法的威脅

未來的量子電腦能夠利用 Shor 演算法，在極短時間內破解目前廣泛使用的非對稱加密（如 RSA、Diffie-Hellman 和橢圓曲線加密）。這些演算法是現今 TLS 網路通訊的基礎。

HNDL：先收集，後解密

Harvest Now, Decrypt Later (HNDL) 這是目前最迫切的威脅。攻擊者現在就可以攔截並儲存加密的網路通訊數據。雖然他們現在無法解密，但等到 10-20 年後量子電腦成熟，他們就能使用未來的算力解開過去的機密。這使得升級密鑰交換（Key Exchange）機制變得極為急迫。

🛡️ NIST 的解方：PQC 標準化

為了對抗量子運算，美國國家標準暨技術研究院 (NIST) 正在推動一系列能抵抗量子攻擊的新演算法。以下卡片展示了 Java 將支援的核心標準，您可以將滑鼠停留在卡片上查看其功能重點。

🔐

ML-KEM (Kyber)

密鑰封裝 (Key Encapsulation)

基於格（Lattice-based）的演算法。主要用於安全地建立和交換對稱密鑰，是防禦「先收集，後解密」(HNDL) 攻擊的關鍵武器。

✍️

ML-DSA (Dilithium)

數位簽章 (Digital Signature)

同樣基於格的演算法。用於驗證身份和確保資料完整性。將取代現有的 RSA 或 ECDSA 簽章，防止未來的偽造攻擊。

#️⃣

SLH-DSA

基於雜湊的簽章

一種完全不同的數學基礎，使用無狀態雜湊函數（Hash-based）。作為基於格演算法的備用方案，提供多樣性的安全防護。

🤝 混合模式與開發者影響

過渡到新加密演算法並非一蹴可幾。了解 Java 如何透過「混合模式」兼顧安全與穩定，以及這些變更對您的應用程式程式碼有何影響。

🔗 為什麼需要「混合模式」？

PQC 演算法雖然理論上能抵抗量子計算，但它們相對較新，尚未經過長時間的實戰考驗（Battle-tested）。

因此，Java（與 IETF 標準）採用了混合方案 (Hybrid Schemes)，將傳統演算法（如 X25519）與 PQC 演算法（如 ML-KEM）結合。

傳統演算法 (如 X25519) ✔️ 防禦現在
+
PQC 演算法 (如 ML-KEM) ✔️ 防禦未來 (量子)
只要其中一個未被破解，通訊就是安全的！
